目标
为管理员提供一个用于维护「产线」信息的配置页面及对应的服务端能力，建立一份全局共享的产线列表，作为 Project 模块中「产线」字段的唯一数据来源。

在本次改动中需完成从「产线维护」到 Project 使用产线的完整链路，包括：

管理员可以在专用页面中对产线进行增删改查。

Project 数据模型与对外数据契约中新增「产线」信息，使 Project 能够关联到某个产线。

所有 Project 创建 / 编辑页面必须支持选择产线，Project 详情页需展示产线信息。

超出本次范围的内容（仅作为范围边界说明）：

不要求在 Project 列表等聚合视图中基于产线进行筛选、排序或统计。

不要求实现产线相关的审计日志、变更历史等增强能力。

不限定具体 UI 形态、技术框架或数据库物理结构，由实现阶段自行决策。

设计约束
产线实体及约束
产线在系统中作为全局共享的配置实体存在，所有 Project 共用同一份产线列表。

产线的业务字段仅包含：

名称：必填，用于在界面展示和在 Project 中选择；

描述：可选，用于对产线用途或范围做补充说明。

产线名称在全局范围内必须唯一：

创建新产线时不得出现与现有产线同名的记录；

编辑产线名称时，同样不得改为与其他产线重复的名称。

不引入启用 / 停用等状态字段，产线仅存在「存在 / 不存在」两种状态。

产线维护权限与行为约束
仅具有「管理员」角色的用户可以：

访问产线维护页面；

调用用于增删改查产线的服务端能力。

非管理员不得通过 UI 或接口对产线进行新增、修改或删除操作；如尝试访问相关能力，应被明确拒绝（例如通过权限错误）。

对于已被任意 Project 引用的产线：

不允许删除该产线；

删除操作必须显式失败，且失败结果不得导致产线记录消失或 Project 中出现无效引用。

仅当产线未被任何 Project 使用时，才允许删除该产线。

Project 与产线的关联约束
每个 Project 至多关联一个产线，且该产线必须来自当前全局产线列表，不允许使用自由输入的任意值。

Project 的对外数据契约中必须包含足以标识和展示产线的信息，至少包括：

可用于唯一标识产线的标识符；

产线名称；
以便前端或调用方能在 Project 详情中展示产线名称，并在编辑时关联到正确的产线记录。

新建 Project 时对产线的要求
新建 Project 时，「产线」为必填字段：

创建表单在提交时必须要求选择一个已存在的产线；

若未选择产线或选择的产线不存在，则创建操作应被拒绝。

新建 Project 创建成功后，该 Project 的产线字段不得为空，且必须指向当前产线列表中的有效产线。

历史 Project 的处理
对于改造前已存在且当前没有产线信息的历史 Project：

在不进行编辑时，允许继续保持无产线状态；

当用户对该 Project 进行编辑并尝试保存变更时，如果该 Project 仍未关联任何产线，则必须在保存前选择一个产线，否则保存应失败。

一旦历史 Project 在编辑过程中补全了产线信息后，后续对该 Project 的约束应与新建 Project 相同，不再允许产线为空。

已有关联产线的 Project 的修改规则
对于已经关联了产线的 Project：

允许将其关联产线从一个产线变更为另一个存在的产线；

不允许在编辑时将产线字段清空（即从有值改为无值）。

任一 Project 在任意时刻都不得引用不存在的产线；若产线被删除，应保证没有 Project 仍引用该产线后再执行删除操作。

Project 侧权限与可见性
具有「创建 Project」权限的用户在创建表单中必须能够选择产线。

具有「编辑 Project」权限的用户在编辑表单中必须能够查看和变更该 Project 的产线（在上述规则允许的范围内），但不得将其置为空。

任何具有访问 Project 详情页面权限的用户，应能在 Project 详情中看到该 Project 的产线名称；可见性与其它基础项目信息（如项目名称）保持一致。

Project 页面范围约束
在所有 Project 创建页面和编辑页面中：

必须存在用于选择产线的控件；

该控件必须基于当前产线列表提供选项，而非自由文本输入。

在 Project 详情页面中必须展示该 Project 当前关联的产线名称。

本次改动不对 Project 列表等聚合视图中是否展示产线做硬性要求，不强制增加基于产线的筛选、排序或分组能力。

验收标准
管理员登录系统后，可以访问产线维护页面，看到当前全局产线列表以及每条产线的名称和描述。

管理员可以在产线维护页面新增产线：名称为必填项，描述为可选；当名称为空或只包含空白字符时，创建被拒绝。

当管理员尝试创建名称与现有产线重复的新产线时，系统拒绝创建，并明确指出是名称冲突导致失败（错误信息文案可自由实现）。

管理员可以编辑已有产线的名称和描述；当新名称与其它产线名称重复时，保存被拒绝，且不会产生重复名称的记录。

当某条产线当前未被任何 Project 使用时，管理员在产线维护页面可以成功删除该产线，删除后该产线在列表及后续 Project 表单中均不再出现。

当某条产线已被至少一个 Project 使用时，管理员尝试删除该产线会失败，失败后该产线仍存在于产线列表中，对已有 Project 的产线显示不受影响。

非管理员用户无法通过 UI 访问产线维护页面，且尝试通过接口调用产线新增、修改或删除能力时会被拒绝（例如返回权限错误）。

新建 Project 的创建表单中包含产线选择控件，选项来源于当前产线列表，不允许用户输入任意文本作为产线。

当用户在新建 Project 表单中未选择任何产线或选择的产线已不存在时，提交将失败，并给出「产线未选择或无效」类的错误提示；当选择有效产线后提交，则创建成功且该 Project 关联到所选产线。

通过任意对外 Project 读取能力（例如 Project 详情接口）获取新建成功的 Project 时，其返回结果中包含能标识产线的字段，并且可以从中得到产线名称，且名称与产线维护页面中的一致。

对于改造前已有但尚未绑定产线的历史 Project，在不进行编辑时仍可正常查看和使用现有功能，不会因为缺少产线而出错。

当用户编辑一个尚未绑定产线的历史 Project 并尝试保存变更时，如果未选择产线，则保存会失败，并提示需要选择产线；当选择了一个有效产线后再保存，则保存成功且 Project 关联该产线。

对于已经绑定了产线的 Project，编辑页面允许用户将其产线从一个产线改为另一个存在的产线；保存后，Project 的产线信息更新为新的产线，且详情页展示为新的产线名称。

对于已经绑定了产线的 Project，在编辑页面中无法将产线字段清空（例如无法选择「无产线」或保存时产线为空），否则应阻止保存并提示错误。

在任一时刻，对外提供的 Project 数据中不存在引用已删除产线的记录；若某产线被删除，则在此之前必须已保证没有 Project 仍引用该产线。

具有创建 / 编辑 Project 权限的用户，在创建和编辑 Project 时均可以正常选择和修改产线；不具备编辑 Project 权限的用户无法通过 UI 修改产线字段。

任何能够访问 Project 详情页面的用户，在详情中可以看到该 Project 当前关联的产线名称。

Project 详情页面在本次改动后如实展示产线信息，但 Project 列表等其他视图即便暂未展示产线，也不影响本次改动的验收。
